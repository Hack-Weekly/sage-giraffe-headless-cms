How To Use It

Our Headless CMS was structured to be used by any costumer in any type of product. Through our thoughfully engineered architecture, we provide a platform for all your needs.

Features

I. User Authentication and Authorization:

  • Registration: Users can register with a username, password, and role.
  • Login: Users can log in with their credentials, and the CMS verifies the username and hashed password.
  • Logout: Users can log out from their current session.

II. User Management:

  • Admin Dashboard: An admin user has access to an admin dashboard.
  • User Listing: Admin users can view a list of all registered users, sorted by the last login timestamp.
  • Add User: Admin users can add new users to the CMS.
  • Update User: Admin users can update the username, password, and role of any user.
  • Delete User: Admin users can delete a user from the CMS.

III. Content Management:

  • Content Dashboard: Authenticated users can access a content dashboard.
  • View Content: Authenticated users can view a list of all contents, sorted by creation date.
  • Add Content: Authenticated users can create new content by providing a title and body.
  • Update Content: Authenticated users can edit the title and body of their existing content.
  • Delete Content: Authenticated users can delete their own content.

IV. Routing and Page Rendering:

  • Index Route: Renders the default index.html template.
  • Login Route: Renders the login page template and handles user authentication.
  • Logout Route: Logs out the user and redirects to the index route.
  • Register Route: Handles user registration and redirects to the admin dashboard upon successful registration.
  • Admin Dashboard Route: Renders the admin dashboard template and displays the list of registered users.
  • Content Dashboard Route: Renders the content dashboard template and displays the list of contents.
  • Add Content Route: Renders the add content template and handles the creation of new content.
  • Update Content Route: Renders the edit content template and handles the updating of existing content.
  • Delete Content Route: Handles the deletion of content.
  • Add User Route: Renders the add user template for admin users to add new users.
  • Update User Route: Renders the edit user template and handles the updating of user details.
  • Delete User Route: Handles the deletion of a user.
  • 404 Route: Renders a custom 404.html template for invalid routes.
